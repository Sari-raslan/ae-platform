================================================================================
TESTING GUIDE
================================================================================

Component Test Flow:

1. Click "Start Style"
   - Runtime starts
   - running: true

2. Click "INTRO1"
   - Queues INTRO1 section
   - Will apply at bar boundary

3. Click "1 Bar" (pulse 16 times)
   - Advances 16 steps (1 bar)
   - Renders INTRO1 events at their steps
   - Applies queued section

4. Click "VAR1"
   - Queues VAR1
   - Will apply next bar

5. Click "D major"
   - Sets chord to "D major"
   - Next pulse will transpose to D

6. Click "1 Bar"
   - VAR1 plays transposed to D major
   - BASS/ACC1 transposed +2 semitones
   - DRUMS unaffected

Section Events:
  - INTRO1: 4 events
  - VAR1: 8 events
  - VAR2: 12 events (busier)
  - FILL1: 4 events (fill pattern)
  - ENDING1: 3 events (closing)

Chord Support:
  - C major (offset 0)
  - D major (offset 2)
  - A minor (offset 9)
  - Custom chords supported via setChord()

================================================================================
ARCHITECTURE
================================================================================

```
StyleRuntimePanel (React Component)
  ├─ createArrangerStyleRuntime()
  │  ├─ createStyleSectionRuntime()
  │  │  └─ 5 sections with phrase events
  │  └─ transposeStyleEvents()
  │     └─ Chord-based transposition
  └─ State:
     - running: boolean
     - bar: current bar number
     - beat: current beat (1-4)
     - step: current step (0-15)
     - chord: current chord name
     - queuedSection: next section to apply
     - renderedEvents: history (64 events)
     - section: section runtime state
```

Data Structures:
- Section: { step, track, note, velocity }
- Rendered Event: { bar, beat, step, chord, events, renderedAt }
- State: { running, bar, beat, step, chord, renderedEvents, section }
